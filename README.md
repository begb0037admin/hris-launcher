## How to Update the Dashboard

### Option 1 — Via GitHub web interface (no Git required)

1. Go to https://github.com/begb0037admin/hris-launcher
2. Click `index.html` → click the pencil ✏️ (Edit) icon
3. Make your changes in the editor
4. Scroll down → add a commit message → click **Commit changes**
5. GitHub Pages rebuilds automatically — live in ~60 seconds

### Option 2 — Upload a replaced file

1. Go to https://github.com/begb0037admin/hris-launcher
2. Click **Add file → Upload files**
3. Drop the updated `index.html` onto the page
4. Add a commit message → click **Commit changes**
5. Live in ~60 seconds

### Option 3 — Via Git (command line)

```bash
cd "C:\Users\admin\Downloads\hris-launcher-deploy"
# make edits to index.html
git add index.html
git commit -m "describe your changes"
git push origin main
```

## Updating Environment Clone Dates

When a non-production environment is refreshed, update the **Cloned:** date in `index.html`.

Search for the environment code (e.g. `UOXZ`, `UOXC`, `UOXU`) and update the `env-date` div with the new clone date.

## Adding or Removing Tiles / Nav Links

- **Production tiles** — find the `<div class="tiles-row">` section in `index.html`. Each tile is an `<a class="tile ...">` element. Add a corresponding CSS class in the `/* ── TILES ── */` style block for the icon background colour.
- **Sidebar nav links** — find the relevant `<div class="nav-group">` inside `<aside class="sidebar">`. Add or remove `<a class="nav-link">` entries as needed.

## Auth — How It Works

There is no authentication on the launcher itself. Each system (Back Office, Staff Portal, etc.) handles its own login — clicking a tile opens that system in a new browser tab as normal.

GitHub repository access requires the `begb0037admin` account. If pushing via Git over HTTPS, a Personal Access Token (PAT) is needed — generate one at https://github.com/settings/tokens (repo scope).

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Dashboard not updating after edit | GitHub Pages CDN cache | Wait 60–90 seconds, then hard-refresh (Ctrl+Shift+R) |
| Tile link broken / wrong URL | URL changed upstream | Edit `index.html`, update the `href` on the relevant tile |
| Nav link broken | URL changed upstream | Edit `index.html`, update the `href` on the relevant nav-link |
| Page shows 404 | GitHub Pages not configured | Repo Settings → Pages → Source: `main` branch, `/ (root)` → Save |
| Git push rejected | PAT expired | Generate a new PAT at https://github.com/settings/tokens and update credentials |
